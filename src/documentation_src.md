Tous les fichiers commençant par "Colab_" correspondent à des notebooks (.ipynb) qui ne tournent que sur Google Colab. Pour les utiliser dans Jupyter Notebook, certaines cellules doivent être supprimées ou adaptées. Les cellules en question sont commentées pour pouvoir être adaptées. Si vous voulez lancer sur Google Colab, il faut modifier le type d'exécution dans "Exécution" en haut à droite, puis sélectionner GPU T4.
- Colab_prompting_tests.ipynb est un notebook qui teste 7 modèles (Qwen-3 0.6B, GPT-oss, Phi-3, Gemma-3 4B, Orca-mini, Gemma 7B, DeepSeek-R1 7B) sur une seule entrée du Wiktionnaire. Le but est de sélectionner les meilleurs modèles pour ensuite les tester sur toutes les données du Wiktionnaire. Chaque modèle est noté sur 6 selon la justesse de ses réponses au prompt. Vous pouvez retrouver la note de chaque modèle, ainsi que des remarques dans le notebook (sous forme de tableau avec des petites étoiles).  
- Colab_Kaikki_tests est un notebook qui teste les modèles GPT-oss et Phi-3 sur des données de Kaikki. Kaikki est une version structurée du Wiktionnaire. Elle inclut les rubriques étymologiques qui nous intéressent pour identifier les formes supplétives. Vous pouvez retrouver les données fr-extract.jsonl.gz à ce lien : https://kaikki.org/dictionary/rawdata.html (il faut descendre avec la souris pour le trouver). Puisque le fichier Kaikki est très lourd, il n'est pas possible de traiter toutes les données d'une traite. Donc nous ne travaillons que sur un chunk de données (un morceau de 10 entrées maximum). En sortie vous obtenez un csv comportant le lemma, la catégorie, la définition, l'étymologie, la langue, la base, le préfixe, le suffixe, les composants et le type de procédé de chaque entrée du chunk.
- Colab_sauvegarde_csv_chunk_par_chunk est un notebook qui permet de sauvegarder les données petit à petit, toutes les 10 entrées. Cela permet de traiter le plus de données possibles avec un temps et un nombre de GPU limités. Vous obtenez un csv similaire au csv de Colab_Kaikki_tests mais sur beaucoup plus de données et avec un prompt différent.
- Colab_few_shot_prompting est un notebook qui teste l'ajout de deux exemples dans le prompt (même prompt que dans Colab_sauvegarde_csv_chunk_par_chunk). En sortie vous obtenez un csv similaire à Colab_sauvegarde_csv_chunk_par_chunk et Colab_Kaikki_tests.

Tous les fichiers commençant par "Grid_" correspondent à des scripts (.py) que l'on fait tourner sur Grid5000. 
- Grid_test_llama_csv est un script qui utilise le même prompt que Colab_Kaikki_tests mais qui sauvegarde un csv petit à petit comme Colab_sauvegarde_csv_chunk_par_chunk. 
- Grid_test_new_prompt_llama est un script adapté de Colab_sauvegarde_csv_chunk_par_chunk pour tourner sur Grid5000 et teste le modèle Llama3.2.
- Grid_test_new_prompt_phi3 est un script adapté de Colab_sauvegarde_csv_chunk_par_chunk pour tourner sur Grid5000 et teste le modèle Phi-3 mini. 
- Grid_test_few_shot_phi3 est un script adapté de Colab_few_shot_prompting pour tourner sur Grid5000 et teste le modèle Phi-3 mini.

Tous les résultats au format csv sont stockés dans le dossier Results/Colab ou Results/Grid.
